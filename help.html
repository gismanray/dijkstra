About The Dijstra's Shortest Path Algorithm

Dijkstra's algorithm finds the shortest path in a graph (network) made of nodes and edges. It was published by Edsger Dijkstra ['daikstrə] in 1959 and has been widely applied to transportation and GIS since then. This web-based demonstration application was developed by Dr. Ray Huang. Although named "shortest path", the algorithm can find paths that minimize any costs including distance, time, money, etc. that are involved in traversing the edges and nodes in the graph.

The Dijkstra's algorithm is a labelling algorithm. Basically, it is an iterative procedure that progressively labels nodes and edges when it searches the graph and minimizes cumulative costs at nodes. Given a graph G = (N, E), N = {x}, E = {e}, where N and E represent a set of nodes and edges respectively, the procedure below finds the shortest path from an origin node s ∈ N to a destination node t ∈ N.

Step 1: initially, all edges and nodes are unlabeled. Assign a number d(x) to each node x to denote the tentative length of the shortest path from s to x that uses only labeled nodes as intermediate nodes. Initially, set d(s) = 0 and d(x) = ∞ for all x ≠ s. Let y denote the last node that was labeled. Label node s and let y = s.

Step 2: for each unlabeled node x, redefine d(x) as follows:

d(x) = min{d(x), d(y) + e(y, x)}

This can be performed efficiently by scanning the forward star of node y since only these nodes will be affected. If d(x) = ∞ for all unlabeled nodes x, then stop because no path exists from s to any unlabeled node. Otherwise, label the unlabeled node with the smallest value of d(x). Also label the edge directed into node x from a labeled node that determined the value of d(x) in the above minimization. Let y = x

Step 3: if node t has been labeled then stop, since a shortest path from s to t has been discovered. This path consists of the unique path of labeled edges from s to t. If node t has not been labeled yet, repeat step 2.

If you feel the math expression hard to apprehend, the following procedure may help. And the procedure can be carried out with a piece of paper and a pencile. A graph (network) is composed of a set of nodes that are connected by edges each carrying a traversal cost (or length). To find the shortest path from an origin node to a destination node,

Step 1: Assign a travel cost 0 to the origin node and infinity (âˆž) to all the rest nodes. Then set the origin node as the start node and begin the following iterative process.

Step 2, If the start node is the destination node, the shortest path is found, stop.

Otherwise,

1) For the start node, find all its neighboring nodes that can be directly traveled to. Let's call them to-nodes.

2) For each to-node, calculate a cumulative cost by adding the cost at the start node and the cost of the edge between the start-node and the to-node.

If the to-node currently carries infinity, label the node with the cumulative cost;

Otherwise (if the to-node carries a cost, i.e., the node has been labelled),

if the new cumulative cost is smaller than the labeled cost at the to-node, relabel the to-node with the new cumulative cost;

otherwise (if the cumulative cost is greater than or equal to the labeled cost at the to-node), just skip this to-node.

3) Once all to-nodes from the start node are labeled, close the start node and label the edge leading to the start node. This marks the end of an iteration of network search.

Step 3: Among all labelled nodes (i.e., nodes carrying a cost but are not closed), find the one that carries the minimum cost and set it as the start node, then repeat step 2. If no start node is found, i.e., all nodes are either closed or unlabeled, the entire network is searched through, stop.

How to Use The Dijkstra Algorithm Learning App

This Dijkstra's algorithm learning app helps you understand the algorithm in a way similar to using pen and paper, but more conveniently. When you open the page, an example graph is loaded. You can use the select tool to relocate nodes by dragging them, and you can adjust the cost of edges by dragging the slider on them. To search the shortest path(s), you must use the set origin tool to set an origin node before start, and you may or may not set a destination. Then you can click the step over tool to move the algorithm step by step, or click the auto run tool once to run through. Network search process is explained step-by-step in the lower box. The following briefly explains the tools on the page.

Select:Use it to move nodes and adjust (slide) edge costs
Add node:Use it to add new nodes by clicking on the map
Add edge:Use it to add edges by dragging from one node to another
Set origin:Use it to set an origin by clicking on a node
Set destination:Use it to set a destination by clicking on a node
Delete:Use it to delete nodes/edges by clicking on the nodes or sliders
Step over:Click it to move one step forward in path search
Auto run:Click it to run through a path search
